<컴퓨터개론 - 김대수>

컴퓨터가 일을 처리하기 위한 대상 : data
데이터를 처리한 결과 : information

* 컴퓨터의 기능
입력 기능 : 처리할 데이터와 프로그램을 외부로부터 읽어들여 기억장치로 전달
기억 기능 : 읽어들인 데이터와 프로그램들과 처리된 결과 등을 기억장치에 기억
연산 기능 : 데이터를 산술연산과 논리 연살을 하여 결과를 만들어 내는 기능
제어 기능 : 프로그램들의 명령을 동작하도록 장치들을 제어하고 지시하는 기능
출력 기능 : 컴퓨터 내부에 기억된 내용과 처리된 결과를 보여주는 기능

* 시스템 소프트웨어
 - 시스템 소프트웨어 : 사용자가 복잡한 하드웨어를 모르고서도 사용할 수 있도록 도와주는 프로그램 종류에는 운영체제, 언어 번역 프로그램, 유틸리티 등
 - 디바이스 드라이버 : 장치 구동기, 하드웨어와 운영체제의 중간에 위치함, 하드웨어 장치들을 동작시키는 일을 담당함
 - 펌웨어 : 시스템의 효율을 높이기 위해 ROM에 들어 있는 기본적인 프로그램

* 운영체제
하드웨어와 소프트웨어 자원들을 관리하고 제어
응용소프트웨어를 실행할 수 있는 기반이 되는 시스템 소프트웨어
Job Scheduler

* 프로세스 : 작업의 단위, 작업 도중 필요한 정보나 스케쥴에 필요한 여러가지 정보를 기억하고 있는 구조체
프로세스 생성 ------> 준비 --------> 실행 -----> 프로세스 종료
                           └----대기<-----┘
                               (인터럽트)
* 버퍼링
입력 -> 입력버퍼 -> cpu -> 출력버퍼 -> 모니터

단방향 통신(Simplex) : 수신기능만 갖고 있음
반이중 통신(Half Duplex) : 한번에 한 방향으로만
전이중 통신(Full Duplex) : 송신과 수신을 동시에

라우터 : 수신되는 인터넷 주소를 보고 경로를 정하는 역할

*데이터종속성(논리종속성)
데이터를 사용하는 프로그램의 구조가 데이터 구조(파일 구조)의 영향을 받는다는 것을 의미
데이터 종속성은 프로그램의 개발과 유지보수를 어렵게 한다

Q. 영업부에 속한 모든 사원의 이름과 전화번호를 보이시오
SELECT ename, tel
FROM employer
WHERE dept='영업'

*키
- 기본키(Primary key) : 각각의 데이터가 다른 데이터와 구분될 수 있는 고유값
- 외래키(Foreign key) : 다른 테이블의 데이터를 참조할 수 있도록 도움을 주는 키

*엔티티
업무의 관심 대상이 되는 정보를 갖고 있거나, 그에 대한 정보를 관리할 필요가 있는 유형

*정규화
ERD내에서 중복요소를 찾아 제거해나가는 과정
- 제1정규화 : 엔티티에서 하나의 속성이 복수개의 값을 가짐
                -> 하나의 속성이 단일 값을 갖도록 설계를 변경
- 제2정규화 : 주식별자가 아닌 속성 중에서 주식별자 전체가 아닌 일부 속성에 종속된 속성을 찾아 제거
- 제3정규화 : 주식별자가 아닌 속성들 중에서 종속관계에 있는 속성을 찾아 제거

상속(Inheritance)
부모가 가진 것을 자식이 가진것과 동일하게 취급

class Person (부모 클래스)

class Student extends Person (자식 클래스) Student is Person
class Teacher extends Person  (자식 클래스) Teacher is Person
class Emp extends person (자식 클래스) Emp is Person

Teacher extends Person
Teacher는 Person을 확장한다

상속 : 부모의 DNA를 자식이 갖는 것
클래스에서의 DNA : 멤버 변수, 멤버 메소드

클래스 전체 : 멤버변수
클래스 내부 메소드 : 지역변수
